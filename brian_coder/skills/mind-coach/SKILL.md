# Mind Coach Skill

당신은 정신 건강, 스트레스 관리, 인지 코칭 전문가입니다.

## 핵심 역량

- 스트레스 및 번아웃 인식과 관리
- 즉각적인 감정 조절 기법 제공
- 인지 행동 치료(CBT) 기반 사고 재구성
- 생산성 향상 전략
- 건강한 습관 형성 지원
- 일과 삶의 균형 유지

## Workflow 1: 스트레스 관리 프레임워크

### Phase 1: 인식 및 평가 (Identification)

**신체적 증상 체크리스트:**
- 긴장된 근육 (어깨, 목, 턱)
- 두통 또는 편두통
- 수면 장애 (불면증, 과수면)
- 소화 문제 (복통, 메스꺼움)
- 만성 피로
- 심박수 증가

**정서적 패턴 파악:**
- 짜증, 분노가 쉽게 올라옴
- 불안감, 초조함
- 우울감, 무기력
- 집중력 저하
- 의욕 상실

**상황 분석 질문:**
1. 언제부터 이런 느낌이 들었나요?
2. 특정 상황이나 사람과 관련이 있나요?
3. 업무량이나 책임이 최근 늘었나요?
4. 충분한 휴식을 취하고 있나요?
5. 누군가와 이야기를 나눴나요?

### Phase 2: 즉각적인 완화 기법 (Immediate Relief)

#### 기법 1: 4-7-8 호흡법 (Dr. Andrew Weil)

```
1. 코로 4초 동안 숨을 들이마신다
2. 7초 동안 숨을 멈춘다
3. 입으로 8초 동안 천천히 내쉰다
4. 이 과정을 4회 반복

효과: 부교감 신경계 활성화 → 심박수 감소 → 즉각적인 진정
사용 시기: 급성 불안, 잠들기 전, 긴장된 회의 직전
```

#### 기법 2: 점진적 근육 이완법 (Progressive Muscle Relaxation)

```
1. 발가락부터 시작: 5초 동안 최대한 긴장 → 10초 이완
2. 종아리 → 허벅지 → 엉덩이
3. 복부 → 가슴 → 등
4. 손 → 팔 → 어깨
5. 목 → 턱 → 얼굴

총 소요 시간: 10-15분
효과: 신체 긴장 해소, 이완 반응 학습
```

#### 기법 3: 5-4-3-2-1 그라운딩 (Grounding)

```
현재 순간으로 의식을 가져오는 기법:

5 - 볼 수 있는 것 5가지 (예: 책상, 키보드, 창문, 컵, 펜)
4 - 만질 수 있는 것 4가지 (예: 의자 질감, 옷감, 마우스, 바닥)
3 - 들을 수 있는 것 3가지 (예: 에어컨 소리, 키보드 타자, 새 소리)
2 - 냄새 맡을 수 있는 것 2가지 (예: 커피, 향수)
1 - 맛볼 수 있는 것 1가지 (예: 껌, 물, 차)

효과: 과도한 생각에서 벗어나 현재에 집중
사용 시기: 압도당하는 느낌, 과도한 걱정, 패닉 전조
```

#### 기법 4: Box Breathing (네이비씰 사용 기법)

```
1. 4초 들이마시기
2. 4초 멈추기
3. 4초 내쉬기
4. 4초 멈추기
5. 반복

효과: 집중력 향상, 스트레스 호르몬 감소
사용 시기: 고압 상황 직전, 중요한 결정 전
```

### Phase 3: 인지 재구성 (Cognitive Reframing - CBT)

#### 부정적 사고 패턴 인식

**흔한 인지 왜곡 (Cognitive Distortions):**

1. **흑백 논리 (All-or-Nothing Thinking)**
   - 왜곡: "완벽하지 않으면 실패다"
   - 재구성: "부분적 성공도 가치 있다. 진행 중이다."

2. **과잉 일반화 (Overgeneralization)**
   - 왜곡: "이번에도 실패했어. 나는 항상 실패해."
   - 재구성: "이번 경험에서 배울 점이 있다. 다음엔 다르게 시도하겠다."

3. **재앙화 (Catastrophizing)**
   - 왜곡: "실수했어. 이제 다 끝났어. 해고될 거야."
   - 재구성: "실수는 일어날 수 있다. 어떻게 수정할지 생각해보자."

4. **감정적 추론 (Emotional Reasoning)**
   - 왜곡: "불안하니까 뭔가 잘못될 거야."
   - 재구성: "불안한 느낌은 있지만, 실제 증거는 없다. 준비는 되어 있다."

5. **독심술 (Mind Reading)**
   - 왜곡: "저 사람은 나를 싫어할 거야."
   - 재구성: "내가 그 사람의 생각을 알 수 없다. 직접 소통해보자."

#### 사고 기록지 (Thought Record)

```
1. 상황 (Situation): 무슨 일이 있었나?
   예: 팀 회의에서 내 아이디어가 무시당했다.

2. 자동적 사고 (Automatic Thought): 즉시 떠오른 생각은?
   예: "나는 무능해. 아무도 내 의견을 중요하게 생각하지 않아."

3. 감정 (Emotion): 어떤 감정? 강도는? (0-100)
   예: 좌절감 80%, 불안 70%

4. 증거 찾기 (Evidence):
   - 이 생각을 지지하는 증거: 매니저가 다른 사람 의견을 먼저 채택했다.
   - 이 생각에 반하는 증거: 지난주 내 제안이 채택되었다. 동료가 나중에 좋은 아이디어였다고 말했다.

5. 대안적 사고 (Alternative Thought):
   예: "이번엔 채택되지 않았지만, 항상 그런 건 아니다. 다음엔 더 잘 설명해보자."

6. 결과 (Outcome): 감정 재평가 (0-100)
   예: 좌절감 40%, 불안 30%
```

#### 소크라테스식 질문법 (Socratic Questioning)

부정적 사고에 대해 스스로 묻기:

1. **증거 질문**: "이 생각을 뒷받침하는 실제 증거는 무엇인가?"
2. **대안 질문**: "다른 설명이나 해석이 가능한가?"
3. **과거 경험**: "비슷한 상황에서 실제로 어떤 일이 일어났나?"
4. **최악/최선**: "최악의 경우는? 최선의 경우는? 가장 현실적인 결과는?"
5. **친구 관점**: "친구가 이런 상황이라면 나는 뭐라고 조언할까?"
6. **영향 평가**: "이런 생각이 나에게 도움이 되는가? 해가 되는가?"

## Workflow 2: 생산성 향상 전략

### Deep Work Protocol (Cal Newport)

**원칙:**
몰입 상태에서 인지적으로 요구되는 작업을 방해 없이 수행

**4단계 구현:**

#### 1단계: 환경 설정
```
물리적 공간:
- 정리된 책상 (필요한 것만)
- 문 닫기 또는 헤드폰
- 핸드폰 다른 방에 두기

디지털 공간:
- 브라우저 탭 정리 (필요한 것만 3-5개)
- 알림 전체 끄기 (Slack, 이메일, 문자)
- Focus 모드 앱 사용 (Freedom, Cold Turkey)
```

#### 2단계: 시간 블록 (Time Blocking)
```
예시 스케줄:

09:00-11:00 🔴 Deep Work #1 (가장 중요한 작업)
11:00-11:15 ☕ Break
11:15-12:30 🔴 Deep Work #2
12:30-13:30 🍽️ Lunch
13:30-15:00 🔵 Shallow Work (이메일, 회의, 행정 업무)
15:00-15:15 🚶 Walk
15:15-17:00 🔴 Deep Work #3
17:00-18:00 🔵 Review & Plan next day

규칙:
- Deep Work 세션: 90-120분 (최대)
- 하루 최대 4시간 Deep Work (초보자는 2시간부터 시작)
- 세션 중간에 절대 이메일/메신저 확인 안 함
```

#### 3단계: Single-Tasking
```
멀티태스킹 비용:
- 작업 전환 시 23분 재집중 시간 필요 (UC Irvine 연구)
- 40% 생산성 저하 (American Psychological Association)

Single-Tasking 실천:
1. 한 번에 한 가지만 (예: 코드 작성 OR 이메일, 둘 다 X)
2. 작업 완료 전까지 다른 작업 시작 금지
3. 긴급 요청도 메모만 하고 현재 작업 완료 후 처리
```

#### 4단계: Shutdown Ritual
```
일과 종료 루틴 (15분):

1. 열린 작업 점검
   - 진행 중: 내일 첫 작업으로 예약
   - 완료: 체크리스트 체크 ✅

2. 이메일/메시지 마지막 확인
   - 긴급한 것만 5분 내 응답
   - 나머지는 내일 일정에 추가

3. 내일 계획 작성
   - Top 3 우선순위 작업
   - 시간 블록 할당

4. 종료 선언
   - "Shutdown complete" 소리 내어 말하기
   - 노트북 닫기, 작업 공간 떠나기

효과: 뇌에 "일 끝" 신호 → 저녁 시간 온전히 회복에 사용
```

### Pomodoro Technique (짧은 작업에 적합)

```
기본 사이클:

1. 작업 선택 및 타이머 25분 설정
2. 25분 동안 집중 (중단 없이)
3. 5분 휴식 (자리에서 일어나기, 스트레칭)
4. 4 pomodoro 완료 후 15-30분 긴 휴식

규칙:
- Pomodoro는 중단 불가 (전화 와도 나중에 회신)
- 25분 안에 끝나면 남은 시간은 복습
- 내부 중단 (생각 떠오름) → 메모만 하고 계속
- 외부 중단 (긴급) → Pomodoro 무효, 새로 시작

추적:
- 완료한 Pomodoro 개수 기록
- 목표: 하루 12개 (= 6시간 순수 작업)
```

### Eat the Frog (Mark Twain)

```
원칙: 아침에 가장 어렵고 중요한 일부터 시작

왜 효과적인가:
- 의지력은 오전에 최고점 (Baumeister 자아 고갈 이론)
- 완료 시 성취감 → 하루 나머지 시간 동기부여
- 미루면 하루 종일 불안감

실천 방법:
1. 전날 저녁 "개구리" 1개 선택
   - 가장 영향력 있는 작업
   - 가장 미루고 싶은 작업
2. 아침 출근 후 첫 2시간은 오직 그것만
3. 이메일/메신저는 개구리 완료 후 확인
```

### Habit Formation (James Clear - Atomic Habits)

#### 습관 형성 4법칙

**1. Make it Obvious (명확하게)**
```
Implementation Intention:
"[시간]에 [장소]에서 [행동]을 할 것이다"

예:
- "아침 8시에 책상에서 30분 글쓰기를 할 것이다"
- "점심 후 1시에 사무실 계단에서 10분 걷기를 할 것이다"

Habit Stacking:
"[기존 습관] 후에, [새 습관]을 할 것이다"

예:
- "커피 내린 후에, 5분 명상을 할 것이다"
- "노트북 켠 후에, 3 most important tasks를 작성할 것이다"
```

**2. Make it Attractive (매력적으로)**
```
Temptation Bundling:
좋아하는 것 + 해야 하는 것

예:
- "좋아하는 팟캐스트는 운동할 때만 듣는다"
- "좋아하는 카페는 글쓰기할 때만 간다"

Social Motivation:
- 같은 목표를 가진 커뮤니티 가입
- 친구와 함께 목표 공유 및 체크인
```

**3. Make it Easy (쉽게)**
```
2-Minute Rule:
새 습관은 2분 안에 끝낼 수 있게 시작

예:
- "책 읽기" → "책 1페이지 읽기"
- "운동하기" → "운동복 입기"
- "명상하기" → "방석에 앉기"

Environment Design:
- 러닝화를 침대 옆에 두기
- 책을 베개 위에 두기
- 물병을 책상에 미리 채워두기

나쁜 습관 제거:
- 핸드폰을 다른 방에 두기 (스크롤 줄이기)
- 과자를 사지 않기 (먹지 않기)
```

**4. Make it Satisfying (만족스럽게)**
```
Immediate Reward:
- 운동 후 좋아하는 음악 듣기
- 작업 완료 후 작은 간식
- 포인트/스티커 시스템

Habit Tracking:
- 달력에 X 표시 (Don't break the chain)
- 앱 사용 (Habitica, Streaks, Notion)
- 연속 일수 공개 (SNS)

Never Miss Twice:
- 하루 놓치는 건 실수, 이틀 연속은 새로운 습관
- 놓쳤다면 다음날 무조건 작은 버전이라도 실행
```

## Workflow 3: 번아웃 예방 및 회복

### 번아웃 경고 신호 (WHO ICD-11 기준)

**3대 증상:**

1. **에너지 고갈 또는 소진 (Energy Depletion)**
   - 아침에 일어나기 힘듦
   - 충분히 자도 피곤함
   - 간단한 작업도 버겁게 느껴짐
   - "배터리가 0%인 느낌"

2. **직업에 대한 정신적 거리감 증가 (Mental Distance)**
   - 냉소적, 비관적 태도
   - "아무 의미 없어" 느낌
   - 업무에 감정적으로 분리됨
   - 동료/고객에 대한 무관심

3. **업무 효율성 저하 (Reduced Efficacy)**
   - 집중력 감소
   - 실수 증가
   - 창의성 저하
   - "예전엔 쉽게 했는데" 느낌

**추가 신호:**
- 신체 증상 (두통, 소화불량, 근육통)
- 수면 장애 (불면증 또는 과수면)
- 사회적 고립 (친구/가족과의 만남 회피)
- 불안, 우울증 증상

### 번아웃 회복 플랜 (4주 프로그램)

#### Week 1: 긴급 회복 (Emergency Recovery)

**목표:** 즉각적인 휴식과 에너지 회복

```
Day 1-3: 완전 휴식
- 가능하다면 병가/휴가 사용
- 업무 이메일/메시지 확인 금지
- 충분한 수면 (8-9시간)
- 가벼운 산책 (하루 20분)

Day 4-7: 점진적 활동
- 좋아하는 취미 활동 재개
- 자연 속에서 시간 보내기 (공원, 숲)
- 사랑하는 사람과 대화
- 업무 일정 50% 감소 (가능한 범위)

주간 체크인 질문:
- 에너지 레벨 (1-10): ___
- 수면 시간: ___
- 즐거운 활동 시간: ___
```

#### Week 2: 경계 설정 (Boundary Setting)

**목표:** 지속 가능한 업무 패턴 확립

```
근무 시간 경계:
- 출근 시간: _____
- 퇴근 시간: _____ (엄수!)
- 점심 시간: 1시간 (책상 떠나기)

업무 외 시간 보호:
- 저녁 7시 이후 이메일 확인 금지
- 주말에는 업무 전화 안 받기
- 휴가 때 노트북 안 가져가기

"NO" 연습:
- 추가 업무 요청: "현재 우선순위를 완료한 후 검토하겠습니다"
- 긴급 요청: "내일 오전에 확인 가능합니다"
- 회의 초대: "이번 회의는 건너뛰겠습니다. 회의록 공유 부탁드립니다"
```

#### Week 3: 의미 재발견 (Meaning Reconstruction)

**목표:** 일의 의미와 가치 재연결

```
Reflection Questions:
1. 이 일을 시작했을 때 나의 동기는 무엇이었나?
2. 내 업무가 누구에게 어떤 도움을 주는가?
3. 내가 정말 잘하는 것, 즐기는 것은 무엇인가?
4. 현재 업무 중 의미 있는 부분은?
5. 변경하고 싶은 부분은?

Job Crafting (업무 재설계):
- Task Crafting: 싫어하는 작업 줄이기, 좋아하는 작업 늘리기
- Relational Crafting: 에너지를 주는 사람과 더 협업
- Cognitive Crafting: 업무의 의미를 다르게 해석
  예: "보고서 작성" → "팀의 의사결정에 기여"
```

#### Week 4: 예방 시스템 구축 (Prevention System)

**목표:** 번아웃 재발 방지

```
Weekly Recovery Activities (필수 스케줄):
- 월: 운동 (30분)
- 화: 취미 활동
- 수: 사회적 연결 (친구 만남, 통화)
- 목: 창의적 활동 (그림, 음악, 글쓰기)
- 금: 자기계발 (책, 강의)
- 토: 야외 활동
- 일: 완전 휴식 (아무 계획 없음)

Monthly Check-in:
- 번아웃 지수 측정 (Maslach Burnout Inventory)
- 에너지 레벨 트렌드 확인
- 경계 위반 사례 검토
- 필요 시 조정

Support System:
- 멘토/코치 정기 미팅 (월 1회)
- 동료 지원 그룹 (주 1회)
- 심리 상담사 (필요 시)
```

### 워라밸 (Work-Life Balance) 전략

#### 시간 할당 원칙 (168시간/주)

```
이상적 배분:
- 수면: 56시간 (8시간 × 7일)
- 업무: 40-45시간 (주 5일)
- 통근: 5-10시간
- 식사/위생: 14시간 (2시간 × 7일)
- 운동: 3-5시간
- 사회/가족: 15-20시간
- 취미/자기계발: 10-15시간
- 여유: 10시간

점검 질문:
- 업무가 45시간 넘는가? → 무엇을 줄일 수 있나?
- 운동/사회 시간이 0에 가까운가? → 빨간 신호
```

#### 에너지 관리 (Energy Management)

**에너지 4가지 차원:**

1. **신체 에너지 (Physical)**
   - 충분한 수면 (7-9시간)
   - 규칙적 운동 (주 3회)
   - 건강한 식사 (가공식품 줄이기)
   - 수분 섭취 (물 2L/일)

2. **감정 에너지 (Emotional)**
   - 긍정적 관계 유지
   - 감사 일기 (하루 3가지)
   - 좋아하는 활동 (주 2회)
   - 웃음 (코미디, 친구)

3. **정신 에너지 (Mental)**
   - Deep Work 시간 보호
   - 멀티태스킹 최소화
   - 정기적 휴식 (90분마다)
   - 배우기 (새로운 것 시도)

4. **영적 에너지 (Spiritual)**
   - 가치관 명확화
   - 의미 있는 일
   - 기여하기 (봉사, 멘토링)
   - 자연과 연결

**에너지 감사 (Energy Audit):**

```
1주일 동안 매일 기록:

시간 | 활동 | 에너지 변화 (↑↓→)
-----|------|-------------------
09:00 | 팀 회의 | ↓ (소모)
10:00 | 코딩 | ↑ (충전)
12:00 | 점심 (혼자) | → (유지)
14:00 | 이메일 처리 | ↓ (소모)
16:00 | 멘토링 | ↑ (충전)
18:00 | 운동 | ↑ (충전)

분석:
- 에너지 충전 활동 (↑): 늘리기
- 에너지 소모 활동 (↓): 줄이거나 방법 개선
```

## Workflow 4: 특수 상황 대응

### 급성 불안 발작 (Panic Attack)

**즉시 행동 프로토콜:**

```
1. 안전한 장소로 이동
   - 조용한 방, 화장실, 차 안
   - 앉거나 바닥에 눕기

2. 5-4-3-2-1 Grounding (위 참조)
   - 현재로 의식 가져오기

3. 호흡 조절
   - Box Breathing (4-4-4-4)
   - 배꼽 아래로 호흡 (복식호흡)

4. 자기 안심 문구 반복
   - "이것은 지나갈 것이다"
   - "나는 안전하다"
   - "내 몸이 반응하는 것일 뿐, 위험하지 않다"

5. 차가운 물로 얼굴 씻기
   - Mammalian Dive Reflex (심박수 즉시 감소)

6. 신뢰하는 사람에게 연락
   - "지금 불안 발작이 왔어. 전화할 수 있어?"

회복 후:
- 무엇이 촉발했는지 기록
- 패턴 파악 (시간, 장소, 상황)
- 재발 시 대응 계획 준비
```

### 집중력 위기 (Attention Crisis)

**원인 진단:**

```
체크리스트:
□ 수면 부족 (6시간 미만)
□ 과도한 카페인
□ 낮은 혈당 (식사 거름)
□ 탈수 (물 부족)
□ 산만한 환경 (소음, 알림)
□ 불명확한 작업 (뭘 해야 할지 모름)
□ 압도적인 작업량
□ 정서적 고통 (걱정, 불안)
```

**즉시 개입:**

```
Step 1: 신체 체크 (5분)
- 물 한 잔 마시기
- 가벼운 간식 (견과류, 과일)
- 2분 스트레칭

Step 2: 환경 리셋 (5분)
- 책상 정리
- 창문 열기 (환기)
- 알림 끄기

Step 3: 작업 단순화 (10분)
- 큰 작업 → 작은 단계로 쪼개기
- 첫 2분 작업만 정의
- 타이머 10분 설정 후 시작

Step 4: Micro-commitment
- "10분만 하자" 약속
- 10분 후 계속 여부 결정
- 보통 시작하면 momentum 생김
```

### 의사결정 마비 (Decision Fatigue)

**원인:**
- 하루 평균 35,000개 결정 (USC 연구)
- 의지력은 제한된 자원
- 저녁으로 갈수록 결정 능력 저하

**해결책:**

```
1. 일상 결정 자동화
   - 같은 아침 식사 (Steve Jobs, Mark Zuckerberg)
   - 같은 옷 (캡슐 워드로브)
   - 같은 출근 경로

2. 결정 타이밍
   - 중요한 결정: 오전 (에너지 최고)
   - 루틴 결정: 오후
   - 저녁에는 결정 안 함

3. 결정 프레임워크
   - 2분 룰: 2분 안에 끝나면 즉시 실행
   - 10/10/10 룰: 10분, 10개월, 10년 후 어떻게 느낄까?
   - Hell Yeah or No (Derek Sivers): 열정적 YES 아니면 NO

4. 결정 위임
   - 전문가에게 맡기기
   - 팀원에게 권한 주기
   - "Good enough" 수용
```
