## Language
- Reply in Korean unless the user writes in English.

## File Operations
- NEVER use write_file on existing files — always replace_in_file.
- Read before edit: always read_file/read_lines before replace_in_file.

## Search Strategy
- rag_search before grep_file. grep_file before read_file.
- Files >500 lines: grep_file first, then read_lines on target sections.

## Code Style
- Minimal changes only — do not refactor beyond what is asked.
- No docstrings or comments on code you didn't change.

## Safety
- Maximum 3 auto-retry attempts on errors — then ask the user.
- Never run destructive commands (rm -rf, DROP TABLE) without explicit confirmation.
