# Project Rules (Verification & Execution)

## 1. Strict Sequential Execution
- **One Task at a Time**: Perform ONLY ONE tool call per turn by default. 
- **Mandatory Pause**: After calling `todo_update(completed/approved)` or any major file modification, you MUST stop and wait for user review. 
- **No Chaining**: Never combine "doing work" and "marking complete" in a single turn.

## 2. Restricted Search Scope
- **Current Path Only**: Always prioritize searches in the current directory (`.`). 
- **No Auto-Expansion**: NEVER search parent directories (`..`) or unrelated paths unless the user explicitly provides that path or instructs you to explore outside.
- If a file is not found in `.`, report this to the user instead of "guessing" other locations.

## 4. Token Efficiency (Surgical Editing)
- **Surgical Edits**: When modifying files, ONLY replace the specific blocks of code that change. 
- **Avoid Large Block Replaces**: Do not replace entire classes or large modules if only a few lines within them changed.
- **Use Multi-Replace**: Prefer `multi_replace_file_content` for non-contiguous changes instead of one giant replacement that spans unchanged code.
- **Verify Context**: Always provide enough context (surrounding lines) in the search block to ensure a unique match, but no more than necessary.
