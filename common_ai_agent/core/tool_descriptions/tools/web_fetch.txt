# Tool: web_fetch

Fetch/summarize content from a specific URL using Cursor CLI.

# Signature
web_fetch(url="https://example.com", formats="markdown", wait_for=3000)

# Description

Fetches and summarizes content from a specific URL through `cursor-agent`.
Returns concise Markdown text. The `formats` and `wait_for` parameters remain
for compatibility, but Cursor controls page loading and output format.

Requires `cursor-agent` on PATH. Firecrawl and Claude are not fallback
backends for web_fetch.

Key Parameters:
- url      - URL to scrape (required)
- formats  - Compatibility hint; Cursor returns Markdown text
- wait_for - Compatibility hint; Cursor controls page loading

# Good Examples

Good: web_fetch(url="https://docs.python.org/3/library/urllib.html")
  Fetch Python urllib documentation

Good: web_fetch(url="https://pcisig.com/specifications", formats="markdown")
  Scrape a specification index page

Good: web_fetch(url="https://example.com/api", formats="html", wait_for=5000)
  Scrape JS-heavy page with longer wait time

# Bad Examples

Bad: web_fetch(url="local_file.v")
  Use read_file() for local files.

Bad: web_fetch(url="")
  Empty URL. Always provide a valid URL.

# When to Use

- Fetch specific documentation page from a known URL
- Read web articles or blog posts referenced by user
- Scrape API documentation for a dependency
- Get current content from a web page
- Read GitHub README or wiki pages

# When NOT to Use

- Local file reading --> Use read_file()
- Searching the web (no URL) --> Use web_search()
- Extracting structured data --> Use web_extract()
- Local code search --> Use rag_search() or grep_file()

# Tool Precedence

- web_fetch > web_search when you know the exact URL
- web_fetch for targeted content; web_search for discovery
- Always prefer local files (read_file) over web fetch

# Error Recovery

- cursor-agent is not reachable
  1. Verify `cursor-agent` is installed and on PATH
  2. Check WEB_CURSOR_TIMEOUT if the request is slow
  3. Fall back to answering from local knowledge only if web access is unavailable

- Fetch failed or No content retrieved
  1. Check if URL is valid and accessible
  2. Try increasing wait_for for JS-heavy pages
  3. Try a different format (html instead of markdown)
  4. The site may block scrapers -- try web_search instead

- Timeout errors
  1. Increase WEB_CURSOR_TIMEOUT environment variable
  2. Try a simpler URL or ask for a targeted section
  3. Try again -- may be transient network issue

- Bot detection / CAPTCHA
  1. Some sites block automated scrapers
  2. Try web_search() instead -- may have cached results
  3. Inform user that site blocks automated access
