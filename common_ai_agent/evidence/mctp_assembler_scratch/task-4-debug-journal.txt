## Findings
### 2026-05-31 - CL self-check failure details
- Source: `mctp_assembler_scratch/model/cl_model_check.json`
- Value: `fl_errors` reported unresolved names `context_id`, `eom`, `payload_data_word`, `payload_byte_strobe`, and `illegal_apb_access`.
- Interpretation: CL self-check only populates `function_model.transactions[].required_fields`; FL self-check passed because it auto-injects unresolved rule names for its own smoke run.
- Refutes/Confirms: Confirms H3 and narrows fix to SSOT transaction `required_fields`, not Python generator logic.

## Root cause (confirmed 2026-05-31)
- Mechanism: `cycle_model.py` self-check builds minimal transactions using only SSOT `required_fields`, so any executable FL output rule that references an undeclared input name becomes an `fl_error` during CL generation.
- Evidence: `task-4-model-chain.txt` failed with CL self-check errors for the five undeclared names; after adding those names to the relevant SSOT `required_fields`, `task-4-model-chain-green.txt` reported `CL self-check passed=True`.
- Toggle proof: Missing fields reproduced failure; adding only the missing fields made the same generation chain pass.
- Fix scope: `mctp_assembler_scratch/yaml/mctp_assembler_scratch.ssot.yaml` transaction required_fields only.
