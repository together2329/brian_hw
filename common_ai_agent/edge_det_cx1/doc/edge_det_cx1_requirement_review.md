### Design decisions recorded
1. 2-flop synchronizer chosen over 3-flop for area/latency tradeoff on this demo IP.
2. Edge detect is purely combinational (no extra pipeline stage) — latency = sync depth.
3. Both rise and fall detectors implemented in the same module for compactness.

### Risk notes
- No CDC analysis required inside module (edge detect operates fully in clk domain).
- sig_in assumed to be stable for >1 clk period to guarantee single detection.
