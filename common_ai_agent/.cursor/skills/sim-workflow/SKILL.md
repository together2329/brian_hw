# Atlas Sim Workflow

`sim` owns real simulation execution and simulation evidence. It is not just a log parser.

## Runtime Workspace

- Workspace: `workflow/sim/workspace.json`
- Command: `workflow/sim/commands/ssot-sim.json`
- Scripts: `workflow/sim/scripts/*`

Common-engine stage:

```text
sim -> WorkflowStageEngine.run_stage("sim", ip)
```

Evidence:

- `sim/results.xml`
- `sim/scoreboard_events.jsonl`
- `sim/sim_report.txt`
- `cov/coverage.json` generated by SSOT coverage summary
- `logs/stage_engine/sim.json`

## Cursor Use

```bash
python3 .cursor/skills/rtl-to-signoff/scripts/rtl_to_signoff.py <ip> --profile dv --execute --from-stage sim --until sim
```

Do not pipe simulator output through truncating filters. Do not edit sim JSON/logs to pass; rerun the stage.
