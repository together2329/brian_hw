# ATLAS · common_ai_agent frontend

A browser-based UI for `common_ai_agent`. The production UI is built by Vite
into `frontend/atlas/dist/` and served by the FastAPI / WebSocket backend
(`src/atlas_ui.py`).

```
brian_hw/common_ai_agent/
├── src/
│   ├── main.py                ← agent loop (unchanged)
│   ├── web_ui.py              ← legacy SSE UI (kept for compat)
│   └── atlas_ui.py            ← NEW: serves this frontend over WS
└── frontend/
    └── atlas/                 ← THIS FOLDER
        ├── index.vite.html    ← Vite main entry template
        ├── lobby.vite.html    ← Vite lobby entry template
        ├── styles.css         ← design tokens + components
        ├── backend.js         ← window.backend (live ws)
        ├── main.tsx           ← Vite main module entry
        ├── lobby-entry.tsx    ← Vite lobby module entry
        ├── data.tsx           ← data adapters
        ├── shared.jsx         ← <Pill>, <Kbd>, ...
        ├── pipeline.tsx       ← pipeline view
        ├── workspace.tsx      ← chat + sidebar (todo / ssot / diff)
        ├── app.tsx            ← top-level shell, theme, routing
        └── dist/              ← generated by `npm run build`
```

## Running

### 1. Install frontend dependencies

```bash
cd frontend/atlas
npm ci
```

Run this after a fresh clone, after deleting `node_modules`, or after
`package-lock.json` changes.

### 2. Build the Vite frontend

```bash
cd frontend/atlas
npm run build
```

Expected output:

```text
dist/index.vite.html
dist/lobby.vite.html
dist/assets/*.js
```

`src/atlas_ui.py` serves only the built Vite frontend. If `dist/` is missing
or only partially built, the browser shows `ATLAS frontend build missing`.

### 3. Start the Atlas browser server

```bash
cd ../..
ATLAS_FRONTEND_MODE=vite python3 src/atlas_ui.py --host 127.0.0.1 --port 8765
```

Then open:

```text
http://127.0.0.1:8765/
http://127.0.0.1:8765/lobby
http://127.0.0.1:8765/healthz
```

Build before starting the server. The `/assets` static route is mounted at
server startup only when `frontend/atlas/dist/assets` exists. If you build
after the server is already running and see `script load failed /assets/...js`,
restart `src/atlas_ui.py`.

### 4. One-shot verification

From `common_ai_agent/`:

```bash
scripts/atlas_vite_e2e_verify.sh
```

This runs type-checking, Vitest, `npm run build`, starts a test Atlas server,
and drives a real browser through the Vite UI with Playwright.

## Troubleshooting

### `script load failed /assets/index-*.js`

The HTML loaded, but the built JS file did not. Check:

```bash
ls frontend/atlas/dist/assets
```

If files are missing, run `npm run build`. If files exist but the server was
started before the build, restart `src/atlas_ui.py`.

### `ATLAS frontend build missing`

Run:

```bash
cd frontend/atlas
npm run build
```

Then restart the Atlas server.

### Build succeeds with Vite warnings

`npm run build` may warn about classic non-module scripts or a large main
chunk. Those are build-quality warnings, not the command needed to browse the
app. The browser path still requires the Vite `dist/` files plus the Atlas
server above.

## Architecture

```
┌────────────────────┐  WS  ┌──────────────────┐
│ Atlas UI (browser) │ ◀──▶ │ atlas_ui.py      │
│  React + JSX       │      │  FastAPI         │
│  window.backend    │      │  bridge          │
└────────────────────┘      └────────┬─────────┘
                                     │ _textual_* callbacks
                                     ▼
                            ┌──────────────────┐
                            │ main.py          │
                            │  ReAct loop      │
                            │  tools, agents   │
                            └──────────────────┘
```

`atlas_ui.py` reuses the same `_textual_*` callback hooks that `web_ui.py`
plugs into, so the agent code in `main.py` stays untouched. Tokens, todos,
context, and cost events flow out as JSON over `/ws/agent`; user prompts
flow back in.

## WebSocket protocol

### Server → client

| `type`         | payload                                          |
| -------------- | ------------------------------------------------ |
| `hello`        | `{frontend, running}`                            |
| `token`        | `{text, cls?}`              streamed agent reply |
| `reasoning`    | `{text}`                    `<thought>` block    |
| `todo_line`    | `{text}`                    todo update line     |
| `flush`        | `{}`                        end-of-block marker  |
| `context`      | `{used, max}`               token-context bar    |
| `cost`         | `{input, cached, output}`   per-turn token cost  |
| `stage`        | `{stage, state}`            pipeline stage event |
| `tool_event`   | `{tool, phase, result?}`    tool-call lifecycle  |
| `agent_state`  | `{running}`                                      |
| `error`        | `{message}`                                      |
| `done`         | `{}`                        turn complete        |

### Client → server

| `type`        | payload      |
| ------------- | ------------ |
| `prompt`      | `{text}`     |
| `interrupt`   | `{text?}`    |
| `run_stage`   | `{stage}`    *(reserved)*  |
| `tool_call`   | `{name, args}` *(reserved)* |

## Notes

- Do not open `dist/index.vite.html` directly with `file://`; absolute paths
  such as `/assets`, `/vendor`, `/backend.js`, `/api`, and `/ws` need the Atlas
  server.
- The Vite HTML templates are `index.vite.html` and `lobby.vite.html`; the
  generated browser files live under `dist/`.
- Letterbox scaling: the design canvas is 1920×1080 and is scaled to fit the
  viewport.
- Light/dark + shell routing live in `app.tsx`.
